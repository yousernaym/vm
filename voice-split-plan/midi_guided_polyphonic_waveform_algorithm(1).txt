MIDI-GUIDED POLYPHONIC WAVEFORM EXTRACTION
===========================================

Goal
----
Create a "good enough" visual waveform for each active MIDI note from a mixed WAV
recording of a single instrument.

The algorithm supports any number of simultaneous notes. It does not attempt exact
source recovery. Instead, it creates plausible per-note waveforms whose sum
approximately reconstructs the original recording.

Core assumptions
----------------
1. The WAV and MIDI describe the same performance and are time-aligned, or can be
   aligned with a single time offset.
2. All notes use the same instrument and therefore have similar harmonic structure.
3. MIDI note intervals are correct.
4. The absolute tuning of the WAV may be unknown.
5. The output is intended primarily for visualization, not studio-quality source
   separation.
6. Reverb, distortion, pitch modulation, and overlapping harmonics may prevent exact
   separation.

Notation
--------
x[t]            Mixed WAV waveform.
X[k, m]         Complex STFT of the WAV.
n               MIDI note number.
f(n)            Estimated fundamental frequency of MIDI note n.
A               Set of active MIDI notes in a time frame.
P_i[k, m]       Predicted magnitude spectrum for note i.
M_i[k, m]       Soft mask for note i.
S_i[k, m]       Estimated complex spectrogram for note i.
s_i[t]          Reconstructed waveform for note i.
H[h]            Shared relative amplitude of harmonic h.
g(d)            Frequency-spread function around a predicted harmonic.
epsilon         Small value preventing division by zero.

Recommended parameters
----------------------
sample_rate             Original WAV sample rate, commonly 44100 or 48000 Hz.
fft_size                4096 for general use.
hop_size                fft_size / 4, for example 1024.
window                   Hann window.
maximum_harmonics       20 to 40, limited by Nyquist frequency.
frequency_spread_cents  20 to 40 cents.
mask_power              1.5 to 2.0.
tuning_search_range     -100 to +100 cents.
tuning_search_step      1 to 5 cents.
minimum_note_duration   20 to 50 ms.
attack_smoothing        5 to 20 ms.
release_smoothing       30 to 100 ms.

High-level algorithm
--------------------
1. Parse the MIDI and create an active-note timeline.
2. Align MIDI time with WAV time.
3. Compute an STFT of the WAV.
4. Estimate the WAV's global tuning offset.
5. Estimate one shared harmonic template for the instrument.
6. For each STFT frame:
   a. Find every active MIDI note.
   b. Predict the harmonic spectrum of every active note.
   c. Estimate the relative loudness of the notes.
   d. Create one soft spectral mask per note.
7. Apply each mask to the original complex spectrogram.
8. Invert each note spectrogram using the inverse STFT.
9. Smooth and normalize the resulting waveforms for display.
10. Optionally verify that the separated waveforms approximately sum to the original.

Detailed procedure
==================

STEP 1: Parse MIDI into note events
-----------------------------------
For every MIDI note-on and note-off pair, store:

    note_event = {
        midi_note,
        start_time_seconds,
        end_time_seconds,
        velocity
    }

Treat note-on with velocity zero as note-off.

Apply sustain-pedal logic if required:
- When sustain is down, delay note endings until the pedal is released.
- For a simple visual approximation, sustain handling may be omitted initially.

Remove or merge events shorter than minimum_note_duration.

Create a function:

    active_notes(time_seconds)

that returns every note event active at the requested time.

STEP 2: Align MIDI and WAV
--------------------------
If MIDI and WAV are already aligned, use zero offset.

Otherwise, estimate one global MIDI-to-WAV time offset:

1. Build a MIDI onset curve:
   - Add an impulse at every MIDI note onset.
   - Weight the impulse by velocity if desired.
   - Smooth the curve over approximately 10 to 30 ms.

2. Build an audio onset curve:
   - Compute spectral flux or short-time energy changes from the WAV.

3. Cross-correlate the two curves.

4. Choose the offset that maximizes the correlation.

Use:

    aligned_midi_time = wav_time - estimated_offset

For a simple implementation, allow this offset to be entered manually.

STEP 3: Compute the audio STFT
------------------------------
Compute:

    X = STFT(x, fft_size, hop_size, Hann_window)

Retain both:
- magnitude: abs(X)
- phase: angle(X)

Use the original complex X later when applying masks.

Each frame m corresponds approximately to:

    frame_time[m] = m * hop_size / sample_rate

STEP 4: Estimate absolute tuning
--------------------------------
MIDI gives exact pitch ratios, but the recording may not use A4 = 440 Hz.

For MIDI note n and candidate tuning offset delta in cents:

    f(n, delta) =
        440 * 2 ^ ((n - 69 + delta / 100) / 12)

Estimate one global delta:

1. Select frames containing one or a few sustained notes.
2. For each candidate delta in the tuning search range:
   a. Predict the first several harmonics of every active MIDI note.
   b. Measure STFT magnitude near those frequencies.
   c. Sum the measured energy.
3. Choose the delta with the largest summed harmonic energy.

A basic score is:

    tuning_score(delta) =
        sum over selected frames m
        sum over active notes i
        sum over harmonics h
            weight[h] * magnitude_near(h * f(i, delta), m)

Suggested harmonic weights:

    weight[h] = 1 / sqrt(h)

This prevents high harmonics from dominating the score.

If the WAV is known to use ordinary equal temperament at A4 = 440 Hz, set delta = 0.

STEP 5: Estimate one shared harmonic template
---------------------------------------------
All notes are from the same instrument, so use one shared relative harmonic profile.

The template is:

    H[1], H[2], ..., H[maximum_harmonics]

where H[h] is the expected relative magnitude of harmonic h.

Preferred estimation method
---------------------------
Use frames where only one MIDI note is active:

1. For each single-note frame:
   a. Determine the note's fundamental frequency.
   b. Read the STFT magnitude near every harmonic.
   c. Divide all harmonic magnitudes by the first harmonic magnitude, or by their sum.
2. Aggregate measurements across notes and frames using the median.
3. Smooth the resulting harmonic sequence.
4. Normalize so that:

       sum(H) = 1

Fallback template
-----------------
If there are no isolated notes, use:

    H[h] = 1 / h

or:

    H[h] = 1 / sqrt(h)

Then normalize.

Optional refinement
-------------------
The instrument's harmonic profile may vary across pitch. For a better approximation,
learn several templates by MIDI register, for example:

- low register
- middle register
- high register

Interpolate between templates for notes between registers.

STEP 6: Predict each active note's spectrum
-------------------------------------------
For every STFT frame m:

1. Find the active note set:

       A = active_notes(frame_time[m])

2. If A is empty, skip the frame or assign it to a residual track.

3. For every active note i:
   a. Compute its fundamental frequency:

          f_i = f(midi_note_i, estimated_tuning_delta)

   b. Create an initially empty predicted spectrum:

          P_i[k, m] = 0

   c. For harmonic h from 1 to maximum_harmonics:
      - harmonic_frequency = h * f_i
      - stop when harmonic_frequency exceeds Nyquist frequency
      - harmonic_strength = H[h]
      - add a smooth peak around harmonic_frequency

A Gaussian spread in cents works well:

    cents_difference =
        1200 * log2(bin_frequency[k] / harmonic_frequency)

    g(cents_difference) =
        exp(-0.5 * (cents_difference / spread_cents)^2)

Then:

    P_i[k, m] += harmonic_strength * g(cents_difference)

Include MIDI velocity:

    velocity_gain = (velocity / 127) ^ velocity_exponent

A useful velocity_exponent is between 1.0 and 2.0.

Then:

    P_i[k, m] *= velocity_gain

STEP 7: Estimate each note's frame amplitude
--------------------------------------------
The predicted spectra describe shape, but each active note also needs a loudness.

Simple method
-------------
For each note i:

1. Read the mixture magnitude near its predicted harmonics.
2. Compute a weighted average.
3. Use this as note amplitude a_i[m].

Example:

    a_i[m] =
        sum over harmonics h
            H[h] * magnitude_near(h * f_i, m)

Normalize amplitudes among active notes if needed.

Better method: non-negative least squares
-----------------------------------------
For frame m, form a matrix whose columns are the predicted note spectra:

    T_m = [P_1[:,m], P_2[:,m], ..., P_N[:,m]]

Solve:

    minimize || magnitude(X[:,m]) - T_m * a_m ||^2

subject to:

    a_m >= 0

where a_m contains one amplitude per active note.

This can be solved with non-negative least squares.

Then scale each note template:

    P_i[:,m] *= a_i[m]

For a "good enough" visualization, the simple method is acceptable. NNLS usually
improves behavior when many notes overlap.

STEP 8: Apply temporal amplitude smoothing
------------------------------------------
Raw frame amplitudes may flicker.

For each MIDI note event, smooth its amplitude curve:

1. Apply a short attack smoothing time.
2. Apply a longer release smoothing time.
3. Force amplitude to zero outside the MIDI note's active interval.
4. Optionally apply a small fade-in and fade-out to prevent clicks.

A one-pole smoother may be used:

    if target > previous:
        coefficient = attack_coefficient
    else:
        coefficient = release_coefficient

    smoothed =
        coefficient * previous
        + (1 - coefficient) * target

STEP 9: Create soft masks for arbitrary polyphony
-------------------------------------------------
Suppose N notes are active in frame m.

For each note i, compute:

    numerator_i[k,m] = P_i[k,m] ^ mask_power

Then:

    denominator[k,m] =
        epsilon
        + sum over all active notes j of numerator_j[k,m]

The note mask is:

    M_i[k,m] =
        numerator_i[k,m] / denominator[k,m]

The masks naturally support any number of simultaneous notes.

A mask_power greater than 1 makes the separation sharper:

- 1.0 gives very soft blending.
- 1.5 to 2.0 is usually suitable for visualization.
- Larger values may create musical noise or unstable waveforms.

Optional residual mask
----------------------
The harmonic models will not explain all energy. Create:

    explained_mask[k,m] =
        min(1, sum_i M_i[k,m])

    residual_mask[k,m] =
        max(0, 1 - explained_mask[k,m])

The residual may contain:
- transients
- breath or bow noise
- percussion-like attacks
- room noise
- reverb
- model error

For visualization, either:
- ignore the residual,
- display it separately, or
- distribute it among active notes according to their masks.

STEP 10: Build one spectrogram per MIDI note event
--------------------------------------------------
Create an empty complex spectrogram S_event for every MIDI note event.

For every frame where that event is active:

    S_event[k,m] = M_event[k,m] * X[k,m]

This preserves the original mixture phase.

If the same MIDI pitch is played again later, keep each note event separate if distinct
visual waveforms are desired. Otherwise, combine all events having the same MIDI pitch
into one track.

STEP 11: Reconstruct each waveform
----------------------------------
For every note event:

    s_event = inverse_STFT(S_event)

Use the same:
- FFT size
- hop size
- window
- centering behavior

as the forward STFT.

Trim or zero the result outside the MIDI event's time range, with short fades at both
ends.

Because the original mixture phase is reused, the result will not be a mathematically
pure isolated note. It should still be suitable for visual display.

STEP 12: Prepare the waveforms for visualization
------------------------------------------------
Raw audio changes too quickly to display clearly over long durations.

Offer two display modes.

A. Detailed waveform
--------------------
Plot the reconstructed audio samples directly for short time windows.

B. Visual envelope
------------------
For longer time ranges:

1. Divide the waveform into display-sized blocks.
2. For each block compute:
   - minimum sample
   - maximum sample
   or:
   - RMS amplitude
3. Plot the min/max envelope or RMS envelope.

Normalize carefully:
- Use one global scale across every note if relative loudness matters.
- Normalize each note separately only if waveform shape matters more than loudness.
- Clearly label which normalization mode is used.

Optional carrier reconstruction for cleaner visuals
---------------------------------------------------
If the separated waveform looks noisy but only a visual representation is needed,
generate a cleaner synthetic waveform using the estimated fundamental and harmonic
template:

    y_i[t] =
        envelope_i[t]
        * sum over harmonics h
            H[h] * sin(2*pi*h*f_i*t + phase_i[h])

Use the separated note's smoothed amplitude as envelope_i[t].

Possible phase choices:
- zero phase for every harmonic
- random but fixed phase per harmonic
- phase sampled from the original STFT near each harmonic

This synthetic waveform is not extracted audio. It is a visually clean approximation
guided by the WAV and MIDI.

For most visualization systems, it is useful to provide both:
1. extracted waveform using STFT masks
2. clean synthetic waveform using the estimated envelope and harmonics

Pseudocode
==========

function extract_note_waveforms(wav, midi):
    audio, sample_rate = load_wav(wav)
    note_events = parse_midi_note_events(midi)

    midi_offset = estimate_or_receive_alignment(audio, note_events)
    shift_note_events(note_events, midi_offset)

    X = stft(
        audio,
        fft_size = 4096,
        hop_size = 1024,
        window = hann
    )

    magnitude = abs(X)
    frequencies = stft_bin_frequencies(sample_rate, 4096)
    frame_times = stft_frame_times(sample_rate, 1024)

    tuning_delta = estimate_global_tuning(
        magnitude,
        frequencies,
        frame_times,
        note_events
    )

    harmonic_template = estimate_shared_harmonic_template(
        magnitude,
        frequencies,
        frame_times,
        note_events,
        tuning_delta
    )

    for each note_event:
        note_event.spectrogram = zeros_like(X)
        note_event.amplitude_curve = zeros(number_of_frames)

    for each frame m:
        time = frame_times[m]
        active = get_active_note_events(note_events, time)

        if active is empty:
            continue

        templates = []

        for each event i in active:
            fundamental = midi_frequency(
                event.midi_note,
                tuning_delta
            )

            template = build_harmonic_spectrum(
                frequencies,
                fundamental,
                harmonic_template,
                spread_cents = 30,
                maximum_harmonics = 30
            )

            template *= midi_velocity_gain(event.velocity)
            templates.append(template)

        amplitudes = estimate_nonnegative_amplitudes(
            mixture_magnitude = magnitude[:, m],
            note_templates = templates
        )

        for each active event i:
            templates[i] *= amplitudes[i]
            event.amplitude_curve[m] = amplitudes[i]

        templates = temporally_smooth_current_frame_values(
            templates,
            active
        )

        powered_templates = [
            template ^ 1.7
            for template in templates
        ]

        denominator = epsilon
        for each powered_template:
            denominator += powered_template

        for each active event i:
            mask = powered_templates[i] / denominator
            event.spectrogram[:, m] = mask * X[:, m]

    outputs = []

    for each event in note_events:
        waveform = inverse_stft(
            event.spectrogram,
            fft_size = 4096,
            hop_size = 1024,
            window = hann
        )

        waveform = zero_outside_event_with_fades(
            waveform,
            event.start_time,
            event.end_time,
            sample_rate
        )

        display_envelope = compute_min_max_or_rms_envelope(
            waveform
        )

        outputs.append({
            "midi_note": event.midi_note,
            "start_time": event.start_time,
            "end_time": event.end_time,
            "waveform": waveform,
            "display_envelope": display_envelope
        })

    return outputs

Simplified first version
========================
For an initial implementation, use this reduced pipeline:

1. Assume MIDI and WAV are already aligned.
2. Assume A4 = 440 Hz.
3. Use H[h] = 1 / h.
4. Build harmonic templates for all active notes.
5. Estimate note amplitude by weighted energy around its harmonics.
6. Create soft masks using mask_power = 1.5.
7. Multiply the original complex STFT by each mask.
8. Invert each masked STFT.
9. Plot a min/max display envelope for each note.

This version is usually sufficient to demonstrate the concept.

Recommended improvements, in order
==================================
1. Estimate global MIDI-to-WAV time alignment.
2. Estimate global tuning offset.
3. Learn the shared harmonic template from isolated-note regions.
4. Replace simple amplitude estimation with non-negative least squares.
5. Learn separate harmonic templates for different pitch ranges.
6. Model attack noise separately from sustained harmonic energy.
7. Add pitch-bend and vibrato tracking.
8. Add a residual or reverberation track.

Known limitations
=================
1. Exact source recovery is impossible where notes share the same frequency components.
2. Octaves and other harmonically related intervals are especially ambiguous.
3. The reconstructed note waveforms may not sum exactly to the original after trimming,
   smoothing, or normalization.
4. Reverb may appear in several note tracks or in the residual.
5. Strong attack transients may be assigned poorly because they are not strictly
   harmonic.
6. If several notes start and stop together, estimating their individual loudness is
   more difficult.
7. MIDI velocity is not a reliable physical amplitude measurement; treat it only as a
   prior.
8. A strongly nonlinear instrument or effect may create frequencies not predicted by
   the harmonic model.

Validation
==========
Use the following checks:

1. Reconstruction check:

       reconstructed_mix = sum(all note waveforms) + residual

   Compare reconstructed_mix with the original WAV.

2. MIDI activity check:
   Every output waveform should be near zero outside its note event.

3. Spectral check:
   Each note waveform should show energy near the note's fundamental and harmonics.

4. Visual stability check:
   Waveform envelopes should not flicker severely between adjacent frames.

5. Polyphony check:
   Test with one, two, and many simultaneous notes.

Suggested output structure
==========================
For each MIDI note event, output:

    {
        event_id,
        midi_note,
        start_time_seconds,
        end_time_seconds,
        estimated_frequency_hz,
        waveform_samples,
        display_min_max_envelope,
        confidence
    }

A simple confidence value can be based on:

    explained_energy_for_note
    --------------------------------
    total_energy_in_predicted_harmonic_regions

Lower confidence should be expected when:
- many notes overlap,
- the notes are harmonically related,
- the harmonic template fits poorly,
- or there is substantial residual energy.

Summary
=======
The MIDI acts as a time-varying map of which fundamentals and harmonics are expected.
The WAV supplies the actual spectral energy and phase. For every frame, construct a
shared-timbre harmonic model for all active notes, estimate their relative amplitudes,
and divide the mixture spectrum among them with soft masks. Apply the masks to the
complex STFT and invert them to obtain one approximate waveform per MIDI note event.

This method is not exact source separation, but it is well suited to producing stable,
plausible, and visually useful per-note waveforms for arbitrary polyphony.
